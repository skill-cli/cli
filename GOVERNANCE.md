# Governance

`skill-cli` is maintainer-led.

## Maintainer Responsibilities

- keep the package buildable and testable
- preserve documented module boundaries
- review changes against the project scope
- keep compatibility notes current
- decide when proposal work is accepted into architecture docs

## Project Scope

The package builds a native `skill` CLI. Supported agent compatibility is
limited to Codex, Claude Code, Cursor, Gemini CLI, and OpenCode.

## Decision Records

Durable decisions belong in `Documentation/Decisions/`. Architecture documents
should describe the current state, not preserve old alternatives inline.
