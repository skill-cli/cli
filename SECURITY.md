# Security

Security-sensitive issues should be reported privately when possible.

## Reporting

If GitHub private security advisories are available for this repository, use
that channel. Otherwise, contact the maintainer before opening a public issue
with exploit details.

Include:

- affected command or code path
- reproduction steps
- expected impact
- any known mitigations

## Scope

Security review is especially important for:

- skill source downloads
- archive extraction
- filesystem writes
- symlink handling
- agent prompt installation
- watch review state
- install lock handling
